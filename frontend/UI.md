## Component Architecture

### 1. MainLayout (`MainLayout.vue`)
The root structural component. It defines the overall layout using PrimeVue Splitter panels:
- **Header**: Contains the application title.
- **Left Sidebar**: Fixed-width panel for the `FolderTree`.
- **Right Content Area**: Fluid area containing the `TopToolbar` and a vertical split between `MediaTable` and the status area.

### 2. FolderTree (`FolderTree.vue`)
Provides a hierarchical view of the input media directories.
- **Functionality**: Loads the directory structure from the backend and allows users to select a folder to process.
- **Store Connection**: Uses `mediaStore` to fetch the tree and update the currently selected directory.

### 3. TopToolbar (`TopToolbar.vue`)
The action center for batch operations and job configuration.
- **Batch Selection**: Multi-language checkboxes that toggle audio/subtitle streams across all files in the current folder.
- **Output Settings**: Input field to specify the destination directory name.
- **Process Button**: Initiates the cleaning job with the current selections.

### 4. MediaTable (`MediaTable.vue`)
The primary interactive area for fine-grained selection.
- **File List**: Lists all media files found in the selected folder.
- **Stream Selection**: Displays checkboxes for every audio and subtitle stream found within each file, labeled by language (e.g., ENG, GER).
- **Inclusion Toggle**: Each row has a master checkbox to include or exclude that specific file from the batch.

### 5. JobProgress (`JobProgress.vue`)
A dedicated feedback section for active processing tasks.
- **Progress Bar**: Displays the overall percentage completion of the current job.
- **Status Text**: Shows the name of the file currently being processed.

### 6. JobLogs (`JobLogs.vue`)
A console-like view for real-time worker output.
- **Live Updates**: Automatically scrolls to the bottom as new log entries arrive via WebSockets/Store.
- **Detail**: Provides specific FFmpeg output and worker status messages for debugging and monitoring.
